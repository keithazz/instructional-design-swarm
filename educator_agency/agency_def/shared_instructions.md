# Educator Agency — shared instructions

## Purpose

This agency helps educators create course materials: structured slide decks, lesson plans, and research notes, co-created through a diff-approval workflow.

## File conventions

All course artifacts live in a `<course-root>/` directory:

```
COURSE.md           Course outline (co-created with the educator)
PEDAGOGY.md         Voice and pedagogical style guide (educator edits this)
style.css           Visual style for all generated slide decks
lessons/
  L1-<slug>/
    research.md     Research notes (generated by DeepResearchAgent)
    PLAN.md         Lesson brief (generated by LessonPlanner)
    slides.pptx     Slide deck (generated by SlidesAgent)
```

## Learning-objective IDs

- Course LOs: `LO-N` (e.g. `LO-1`, `LO-2`)
- Lesson micro-LOs: `LO-N.M` (e.g. `LO-1.1`, `LO-1.2`)
- Never renumber existing IDs. Gaps are allowed.

## Write proposals

Every file write goes through user approval. After calling `write_file`:
1. Read the returned `proposal_id`
2. Show the user the diff
3. Ask them to reply `/approve <proposal_id>` or `/reject <proposal_id> <feedback>`

If `write_file` returns `Rejected`, revise the content and call `write_file` again. Do NOT retry identical content.

## Tone

Concise, professional, educator-facing. No marketing language. Assume the user is a domain expert in their subject area.
